Prompt Optimization Techniques

Chain-of-Thought (CoT) Reasoning

Break down complex problems into step-by-step reasoning for better accuracy

Without CoT 
Calculate the total cost:
Product A: $45
Product B: $67
Tax: 18%
Discount: 10%

May produce incorrect calculations

With CoT 
Calculate step by step:
1. Sum products: $45 + $67
2. Apply 10% discount
3. Add 18% tax
4. Show final total

Think through each step.

95% accuracy improvement

Few-Shot Learning

Provide examples to guide the model's output format and style

Zero-Shot 
Extract entities from:
"Apple released iPhone 15 
in Cupertino last September"

Inconsistent output format

Few-Shot 
Extract entities as JSON:

Example: "Tesla opened in Austin"
{"company":"Tesla","loc":"Austin"}

Now extract from:
"Apple released iPhone 15..."

Structured, consistent output

3. Role-Based Prompting

Assign specific expertise roles to improve response quality

Generic 
Explain quantum computing.

Generic, unfocused response

Role-Based 
You are a senior quantum physicist.
Explain quantum computing to a 
software engineer learning the field.
Use analogies from classical computing.

Targeted, relevant explanation

Key Best Practices for Llama 4

  • • Be Specific: Define output format, length, and style requirements upfront
  • • Use Delimiters: Separate instructions, examples, and input with clear markers (###, """, ---)
  • • Iterate Context: For multi-turn, maintain conversation context efficiently
  • • Test Temperature: Use 0.7-0.9 for creative tasks, 0.1-0.3 for factual/analytical tasks
  • • Leverage System Messages: Set behavior and constraints at the system level for consistency

Model Selection Decision Tree

Choose Scout 17B when:
  • Response time < 100ms is critical (chatbots, real-time assistance)
  • Deploying to edge devices or mobile
  • Task is well-defined and doesn't require deep reasoning
  • Cost optimization is priority (4x cheaper than Maverick)
  • Use cases: Customer support, code completion, content moderation
Choose Maverick 400B when:
  • Complex multi-step reasoning required
  • Handling multimodal inputs (text + images)
  • Need highest accuracy for critical decisions
  • Research, analysis, or expert-level domain knowledge
  • Use cases: Medical diagnosis assistance, legal document analysis, scientific research
Hybrid Approach:

Use Scout for initial triage/classification, then route complex queries to Maverick. This optimizes both cost and performance.

Multimodal Prompt Patterns

Llama 4 features native multimodal capabilities with early fusion architecture, processing text and images together from the start.

Image + Text Analysis

Pattern Structure:
System: You are an expert image analyst.

User: [IMAGE: product_photo.jpg]

Analyze this product image and provide:
1. Primary objects and their positions
2. Quality assessment (lighting, focus, composition)
3. Potential improvements
4. Recommended use case

Format output as JSON.
Use Cases:
  • E-commerce product quality control
  • Medical imaging preliminary screening
  • Content moderation with context
  • Document understanding (invoices, receipts)

Visual Question Answering (VQA)

Pattern Structure:
System: Answer questions about images accurately.

User: [IMAGE: warehouse_floor.jpg]

Questions:
1. How many pallets are visible?
2. Are safety protocols being followed?
3. What's the estimated occupancy %?
4. Any immediate concerns?

Be specific and cite visual evidence.
Best Practices:
  • Ask specific, measurable questions
  • Request evidence citation from the image
  • Use for inspection, compliance, inventory tasks
  • Combine with CoT for complex reasoning

Multi-Image Comparison

Pattern Structure:
System: Compare images systematically.

User: 
[IMAGE_1: before.jpg]
[IMAGE_2: after.jpg]

Compare these images and identify:
1. Key differences
2. Changes in quality/state
3. Actionable insights
4. Recommendation

Structure: Before | After | Analysis
Applications:
  • Quality control before/after manufacturing
  • Medical progress tracking
  • A/B testing visual designs
  • Damage assessment for insurance

Multimodal Optimization Tips

DO
  • - Provide clear context for each image
  • - Specify expected output format
  • - Use Maverick for complex visual reasoning
  • - Combine images with structured questions
DON'T
  • - Assume model sees everything humans see
  • - Use low-resolution images (<512px)
  • - Mix unrelated images without context
  • - Expect perfect OCR on handwriting

Fine-Tuning Decision Tree

Full Fine-Tuning
Update all model parameters. Best for:
  • Domain-specific language (medical, legal)
  • Large labeled datasets (>10K examples)
  • When prompt engineering fails
Cost: High | Time: Days | GPU: 8x A100
LoRA (Low-Rank Adaptation)
Efficient parameter updates. Best for:
  • Task-specific adaptations
  • Moderate datasets (1K-10K examples)
  • Quick iteration cycles
Cost: Medium | Time: Hours | GPU: 2x A100
Prompt Engineering + RAG
No model updates. Best for:
  • Rapidly changing information
  • Company-specific knowledge bases
  • Limited labeled data (<1K examples)
Cost: Low | Time: Minutes | GPU: Optional
